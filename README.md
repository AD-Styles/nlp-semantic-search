## 📌 프로젝트 요약 (Project Overview)
Sentence-Transformers와 코사인 유사도(cosine-similarity) 를 활용하여 텍스트의 문맥을 분석하고 최적의 결과를 매칭하는 <br>의미 검색(Semantic Search)
1. **[속담 시험출제기]** - 특정 상황을 묘사하는 텍스트를 입력하면 해당 맥락과 가장 잘 어울리는 속담을 찾아 매칭하는 시스템
2. **[유사 상황 대화 검색기]** - 사용자의 질문이나 감정 상태를 입력받아 기존 대화 데이터베이스에서 가장 유사한 문맥을 가진 대화를 찾아내는 검색기

## 🚨 개념 및 파이프라인 (Concept & Pipeline)
| 항목 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; | 세부 내용 |
| :--- | :--- |
| **핵심 개념**<br>**(유사도)** | 다차원 벡터 공간에 매핑된 두 문장 벡터가 이루는 각도를 계산합니다. 각도가 0도에 가까울수록(유사도 1) 두 문장의 의미가 완벽히 일치한다고 판단 |
| **사용**<br>**알고리즘** | `paraphrase-multilingual-MiniLM-L12-v2` 모델 (문장 임베딩), `cosine_similarity` (유사도 연산) |
| **검색**<br>**<nobr>파이프라인</nobr>** | 1. DB 텍스트 사전 임베딩 및 벡터 저장<br>2. 사용자 입력(Query) 실시간 임베딩 변환<br>3. Query 벡터와 DB 벡터 간 코사인 유사도 일괄 계산<br>4. 유사도 기준 내림차순 정렬 후 Top-K 결과 반환 |

## 💡 회고록 (Retrospective)
&emsp;이번 프로젝트를 통해 텍스트를 고정된 클래스로 분류하던 이전 방식을 넘어, 다차원 벡터 공간에서 두 문장 간의 '각도(코사인 유사도)'를 계산하여 가장 가까운 이웃을 찾아내는 검색(Retrieval) 모델의 본질을 깊이 체득할 수 있었습니다. 텍스트 임베딩 모델이 문맥을 이해하는 방식과, 이를 실제 서비스(속담 매칭, 대화 검색)에 적용했을 때 발생하는 엔지니어링적 과제들을 경험했습니다. 특히, 겹치는 단어가 전혀 없더라도 문장이 품고 있는 의미적 맥락이 유사하면 벡터 공간상에서 서로 가까운 좌표에 매핑되어 성공적으로 결과를 도출해 내는 과정을 보며 임베딩(Embedding) 기술의 강력함을 체감했습니다.
<br>&emsp;하지만 실제 구어체 대화 데이터를 다루어 보니, 문장의 전체적인 뉘앙스보다 특정 명사(예: '날씨', '회사')에 벡터의 방향이 과도하게 끌려가는 '키워드 편향' 현상을 목격하며 단순 문장 임베딩의 한계점을 발견했습니다. 또한 입력된 질문과 데이터베이스 내의 모든 문장을 일일이 대조하여 코사인 유사도를 연산하는 현재의 선형 탐색(Linear Search) 방식은, 데이터 크기에 비례해 연산량이 증가하여 대규모 환경에서는 심각한 속도 저하와 병목을 일으킨다는 구조적 한계에 직면했습니다.
<br>&emsp;위와 같은 한계점을 토대로 단순히 가장 유사한 문서를 찾아주는 것을 넘어, 검색된 문맥(Context)을 대형 언어 모델(LLM)에 주입하여 사용자 질문에 대한 맞춤형 답변을 직접 생성해 내는 RAG(검색 증강 생성) 시스템으로 파이프라인을 고도화하는 목표를 가지게 되었습니다.
